# Raw sources — on-policy distillation & the 2026 self-distillation wave

> Provenance note: the 2026 arXiv papers below were read by sub-agents off the arXiv **HTML** (`/abs` + `/html`), not primary PDF reads. Quotes are verbatim-from-the-agent-extraction; exact decimals/equations should be confirmed against the PDF before external citation. The two blogs were read from raw page text (one agent flagged that the first WebFetch pass had fabricated/garbled quotes — those were rebuilt from raw and are the ones kept here).

## S1 — OPD / GKD (the origin)

**"On-policy Distillation of Language Models: Learning from Self-Generated Mistakes"** — Agarwal, Vieillard, Zhou, Stanczyk, Ramos, Geist, Bachem (Google DeepMind / Mila / U. Toronto). arXiv:2306.13649 (v1 Jun 2023, v3 Jan 2024). ICLR 2024.
https://arxiv.org/abs/2306.13649

- Problem (abstract): the *"distribution mismatch between output sequences seen during training and those generated by the student during inference"* (exposure bias).
- GKD generalizes KD on **two axes**: data source (λ) and divergence (D).
- Boxed loss (§3.1): `L_GKD(θ) = (1−λ)·E_{(x,y)~(X,Y)}[D(p_T‖p_S^θ)(y|x)] + λ·E_{x~X}[E_{y~p_S(·|x)}[D(p_T‖p_S^θ)(y|x)]]`
- Token-level (Eq. 2): `D(p_T‖p_S^θ)(y|x) = (1/L_y) Σ_n D(p_T(·|y_<n,x) ‖ p_S^θ(·|y_<n,x))`
- JSD(β) (Eq. 1) interpolates forward↔reverse KL. λ=0 ⇒ supervised KD; λ=1 ⇒ fully on-policy.
- *"we do not backpropagate through the student's sampling distribution p_S(·|x), similar to on-policy imitation."*
- Cost: student sampling adds ≈1.8–2.2× over fixed-dataset draws (Appendix A.2, GSM8K).
- Results: ~2.1× rel. gain summarization (XSUM); ~1.9× GSM8K; MT WMT en→de; FLAN-T5 instruction tuning.

## S2 — Thinking Machines blog

**"On-Policy Distillation"** — Kevin Lu et al., Thinking Machines, Oct 27 2025.
https://thinkingmachines.ai/blog/on-policy-distillation/

- 3-cell taxonomy (verbatim table): SFT = off-policy + dense; RL = on-policy + sparse; on-policy distillation = on-policy + dense.
- *"The core idea of on-policy distillation is to sample trajectories from the student model and use a high-performing teacher to grade each token of each trajectory."*
- Per-token **reverse KL**, expectation over student samples; discount factor 0 (*"the student only optimizes the immediate next token"*). Teacher = a single forward pass.
- *"reinforcement learning only teaches O(1) bits per episode. In contrast, distillation teaches O(N) bits per episode, where N is the number of tokens."*
- *"on-policy distillation can learns [sic] the RL-trained policy in approximately 7-10x fewer gradient steps, which corresponds to a compute efficiency of 50-100x."*
- Qwen3 Technical Report Table 21 (quoted by TM, not measured by them): AIME'24 — off-policy distill 55.0%; +RL 67.6% (17,920 GPU-hr); +on-policy distill 74.4% (1,800 GPU-hr).
- Extends Agarwal 2023 (S1) and Gu et al. MiniLLM; defers loss-choice analysis to S1.
- CAUTION: TM's 9–30× (vs SFT-2M, FLOPs) and 50–100× (vs RL teacher) are *different* comparisons; don't conflate. The ~10× GPU-hr figure is quoted from the Qwen3 report.

## S3 — Privileged-information distillation blog (theory survey)

**"Privileged Information Distillation and Self-Distillation"** — Penaloza, Vattikonda, Gontier, Lacoste, Charlin, Caccia (companion to arXiv:2602.04942, 2026).
https://emilianopp.github.io/Privileged-Information-Distillation-and-Self-Distillation/

- Privileged info `I` = context available at **train** time, not deploy: *"successful agentic trajectories," "ground truth answers," "self-reflection."* Teacher `π_T(y|x,I)`, student `π_S(y|x)`. *"Both start as the same model"* ⇒ self-distillation.
- SFT = forward KL `D_KL(π_T‖π_S)`; self-distillation = reverse KL `D_KL(π_S‖π_T)` (teacher is target); reward-tilted target `π* ∝ π_T·exp(R/β)` ⇒ `max_φ E[R] − β·D_KL(π_S‖π_T)` (*"same form as RL-as-inference, with the teacher as the prior"*).
- KL is over **full-sequence policies**, not per-token. NO hint-injection algorithm, NO error-localization, NO "no-rollout/dense-per-token" claim. (See correction in wiki page.)

## S4 — OPSD / Self-Distilled Reasoner

**"Self-Distilled Reasoner: On-Policy Self-Distillation for Large Language Models"** — Zhao, Xie, Liu, Huang, Pang, Chen, Grover. arXiv:2601.18734 (v1 Jan 26 2026, v3 Mar 20 2026).
https://arxiv.org/abs/2601.18734

- Single model; teacher gets verified solution `y*` in context, student sees only `x`. *"Note that the teacher won't be generating tokens—rationalization is done implicitly through one forward pass."*
- `L_OPSD(θ) = E_{(x,y*)} E_{ŷ~p_S(·|x)} Σ_n D(p_T(·|x,y*,ŷ_<n) ‖ p_S(·|x,ŷ_<n))`, **forward KL**.
- New student rollout each step. No external teacher, no reward model.
- Math only (AIME24/25, HMMT25; Qwen3 1.7B–8B). Qwen3-8B: OPSD 64.8 avg vs GRPO 64.0; Qwen3-1.7B: OPSD 43.4 vs GRPO 37.7 (+5.7). 1024 tokens/sample vs ~16k; ~100 steps vs ~500.

## S5 — SDPO / RL via Self-Distillation

**"Reinforcement Learning via Self-Distillation"** — Hübotter, Lübeck, Behric, Baumann, Bagatella, Marta, Hakimi, Shenfeld, Kleine Buening, Guestrin, Krause. arXiv:2601.20802 (Jan 28 2026; v2 Feb 16 2026).
https://arxiv.org/abs/2601.20802

- Self-teacher = current policy conditioned on **rich textual feedback** `f` (error msgs, test output) that scalar RL discards. *"the self-teacher, π_θ(·|x,f) ... the current policy (the 'student') prompted with the question x and the rich feedback f."*
- `L_SDPO(θ) = Σ_t KL(π_θ(·|x,y_<t) ‖ stopgrad(π_θ(·|x,f,y_<t)))` — per-token, **forward KL**.
- *"Whereas GRPO assigns a constant advantage to each generated token, SDPO assigns an individual advantage to each possible next token ... based on the agreement of student and teacher."* No new rollouts (reuses GRPO's); teacher = parallel forward pass.
- Results: sci-reasoning 70.2 vs 66.6 GRPO (~6× speedup); LiveCodeBench v6 48.8 vs 41.2 (~4×).

## S6 — SDFT / Self-Distillation Enables Continual Learning

**"Self-Distillation Enables Continual Learning"** — Shenfeld, Damani, Hübotter, Agrawal (MIT Improbable AI / ETH). arXiv:2601.19897 (Jan 27 2026).
https://arxiv.org/abs/2601.19897

- SFT on demonstrations is off-policy ⇒ drives catastrophic forgetting. SDFT teacher = same model conditioned on the **demonstration** `c` in context (weights tracked via EMA of student), student sees only `x`. **Reverse KL** on student-generated rollouts.
- *"SDFT leverages in-context learning by using a demonstration-conditioned model as its own teacher, generating on-policy training signals that preserve prior capabilities while acquiring new skills."*
- Results: Sci-QA 70.2 vs 66.2 SFT (retention 64.5 vs 53.4); Tool-use 70.6 vs 63.2; sequential 3-skill accumulation with no regression under SDFT (SFT degrades).

## S7 — SAGE / Self-Hinting (the true source of the "hint" mental model)

**"Self-Hinting Language Models Enhance Reinforcement Learning"** — Liao, Dong, Xu, Monz, Bian. arXiv:2602.03143 (Feb 3 2026).
https://arxiv.org/abs/2602.03143

- SAGE = "self-hint aligned GRPO with privileged supervision." Under sparse terminal rewards GRPO stalls (rollouts in a group get identical reward ⇒ advantages collapse). Model samples a compact **hint (a plan/decomposition)**, then solves conditioned on hint+prompt; task reward unchanged; hints increase within-group outcome diversity so advantages don't vanish.
- This is the "inject a hint to reshape the rollout distribution" idea — closest to the "insert hint tokens" description — but the hint is self-sampled up front, NOT an error-localized patch inserted by a separate critic model.
